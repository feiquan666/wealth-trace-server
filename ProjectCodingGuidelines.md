# 项目规范与编码约定

## 命名规则
- 类名使用驼峰命名法（CamelCase），首字母大写。
- 变量名和方法名使用小驼峰命名法（camelCase）。
- 常量全大写，并用下划线分隔单词。

## 代码格式化
- 使用 IntelliJ IDEA 默认格式化设置。
- 行长度不超过 120 个字符。
- 缩进使用 4 个空格。

## 注释
- 所有公共类、方法和属性都应附带 Javadoc 注释。
- 对于复杂的逻辑块，添加必要的注释以解释其目的。

## 异常处理
- 尽可能捕获具体的异常类型。
- 在抛出异常时，提供有用的错误消息。

## 日志
- 使用 SLF4J 作为日志门面。
- 根据日志级别合理记录信息。

## 测试
- 单元测试覆盖率至少达到 80%。
- 使用 JUnit 5 编写测试用例。
- 每个功能模块都应有对应的测试类。

## 提交信息
- 遵循 AngularJS Git Commit Message Conventions。
- 示例：`feat(api): add new endpoint for user registration`

## 依赖管理
- 使用 Maven 管理项目依赖。
- 定期更新依赖版本以保持最新状态。

以上为初步整理的内容框架，请根据实际情况进行调整和完善。
